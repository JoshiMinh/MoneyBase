# MoneyBase

MoneyBase is a Flutter application that targets multiple platforms, including the web.

## Ensuring up-to-date Flutter Web deployments

Browsers can aggressively cache Flutter Web builds. When redeploying updated builds,
use the following configuration to make sure users receive the latest version without
needing a hard refresh:

### 1. Versioned filenames

Each `flutter build web` invocation generates new hashed assets (for example
`main.dart.js`, `flutter_service_worker.js`, and related files). Because the
filenames change, browsers naturally fetch the new versions after each deploy.

### 2. Cache headers

Configure your hosting provider (Firebase Hosting, Vercel, Nginx, etc.) to serve
`index.html` with `Cache-Control: no-store` (or an equivalent directive). This keeps
the application shell fresh while still allowing long-lived caching for the hashed
assets generated by Flutter.

Together, these settings ensure users always load the latest Flutter Web app without
requiring a manual hard refresh.
